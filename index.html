Asymmetrische Verschlüsselung — Merkwörter

Der private Schlüssel besteht aus 16 Wörtern plus einem Prüfwort. Aufschreiben genügt als Sicherung — aus den Wörtern wird das komplette Schlüsselpaar auf jedem Gerät identisch wiederhergestellt. Alles läuft lokal, nichts wird übertragen.

Nachricht verschlüsseln

Öffentlichen Schlüssel des Empfängers einfügen, Nachricht schreiben. Das Ergebnis landet automatisch in der Zwischenablage.

oder Bild / GIF / Datei

Für Bilder und GIFs wäre der Text zu lang. Stattdessen wird eine PDF zum Herunterladen erzeugt: Sie zeigt beim Öffnen nur einen Hinweis, trägt die verschlüsselten Daten aber im Anhang. Diese PDF lässt sich über Messenger oder Mail teilen; entschlüsselt wird sie unten wieder zum Originalbild.

Nachricht entschlüsseln

Empfangene verschlüsselte Nachricht einfügen — entschlüsselt wird mit deinem Schlüssel von oben.

oder verschlüsselte PDF / Datei

Die geteilte PDF (oder eine sonstige verschlüsselte Datei) hier auswählen. Enthält sie ein Bild, erscheint es als Vorschau mit Speichern-Link.

Speichern
Ableitung: 16 Wörter kodieren 128 Bit Zufall (eigene 256er-Wortliste, nicht BIP39-kompatibel); Wort 17 ist eine Prüfsumme gegen Schreibfehler. Daraus per HKDF-SHA-256 der private P-256-Skalar, der öffentliche Punkt wird lokal berechnet. Nachrichten: ECIES-Schema, Container version ‖ ephemerer Pubkey ‖ Nonce ‖ Chiffrat+Tag, 94 Bytes Overhead. Bilder/GIFs: dasselbe Verfahren, MIME-Typ und Dateiname werden mitverschlüsselt; der Container hängt hinter %%EOF in einer Cover-PDF, die selbst nur einen Hinweistext zeigt. Kompatibel mit sich selbst auf allen Geräten, nicht mit PGP/age.
